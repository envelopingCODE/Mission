# Gamified Task Webb App

## Workflow

- After completing a significant change (a feature, a fix, or a set of related fixes), commit and push to `origin/main` with a clear, descriptive commit message — proactively, without waiting to be asked each time.
- Commit messages: short imperative subject line; add a bullet-point body when the change has multiple distinct parts. Match the existing log style (`git log --oneline`).
- No package, dependency, or CDN may be hosted by Google or Meta/Facebook, and nothing may feed telemetry/usage data back to any big-tech company. Open-source, audited libraries are fine even if big-tech-authored (e.g. React) as long as they're served from a neutral host and send no telemetry — see memory `rule_no_bigtech_telemetry` for full detail. The product should stay safe to offer EU users under GDPR.
